# AutoHotkey 脚本集合

本仓库包含一系列实用的 AutoHotkey v2.0 脚本，用于增强 Windows 操作系统的使用体验。

## 脚本列表

### 1. monitorwiseAltTab.ahk
**显示器感知的 Alt-Tab 窗口切换器**

这是一个增强的 Alt-Tab 实现，仅在当前显示器内切换窗口，避免在多显示器环境中切换时跨越屏幕。

#### 功能特点：
- **显示器感知**：只显示当前鼠标所在显示器的窗口
- **美观的 GUI**：显示窗口图标和标题列表
- **毛玻璃效果**：Windows 10/11 风格的亚克力透明效果
- **圆角设计**：现代化圆角窗口设计
- **智能窗口过滤**：自动过滤工具窗口、隐藏窗口和系统窗口
- **支持最小化窗口**：能够识别并显示最小化窗口的恢复位置

#### 使用方法：
- 按下 `Alt + Tab` 打开窗口切换器
- 保持 `Alt` 键按下，按 `Tab` 键在窗口间循环选择
- 释放 `Alt` 键切换到选中的窗口
- 或直接点击列表中的窗口标题

#### 技术细节：
- 使用 `GetWindowPlacement` API 获取窗口位置信息
- 通过 DWM API 实现毛玻璃效果
- 支持多显示器坐标计算
- 自动清理资源，防止内存泄漏

### 2. mouseButton.ahk
**鼠标侧键增强脚本**

将鼠标侧键（前进/后退键）与滚轮组合，实现更多快捷功能。

#### 功能映射：

| 组合键 | 功能 |
|--------|------|
| **前进键（XButton2）** | 按下：发送 `Shift + NumLock`；松开（无滚轮操作）：触发原前进功能 |
| **前进键 + 滚轮** | 横向滚动（需要应用程序支持） |
| **后退键（XButton1）** | 按下：无操作；松开（无滚轮操作）：原后退功能 |
| **后退键 + 滚轮上** | `Shift + F3` |
| **后退键 + 滚轮下** | `F3` |

**注意**：原功能（前进/后退导航）在松开侧键且未使用滚轮时触发。

#### 设计目的：
- 充分利用鼠标侧键，减少键盘操作
- 防止单独按下 Shift 键时意外触发输入法中英文切换
- 为常用操作（如 IDE 中的查找下一个/上一个）提供快捷方式

#### 注意事项：
- 需要鼠标具备侧键（前进/后退）
- 横向滚动功能取决于应用程序的支持

### 3. rightClick.ahk
**快速右键点击脚本**

最简单的脚本，将 `Ctrl + Space` 映射为右键点击。

#### 使用场景：
- 快速打开上下文菜单
- 在触摸板或特殊键盘上替代鼠标右键
- 提高操作效率

## 安装与使用

### 系统要求
- Windows 10 或更高版本
- AutoHotkey v2.0 或更高版本

### 安装步骤
1. 安装 AutoHotkey v2.0（从 [autohotkey.com](https://www.autohotkey.com/) 下载）
2. 克隆或下载本仓库到本地目录
3. 运行任意脚本：
   - 双击 `.ahk` 文件运行
   - 或右键选择 "Run Script"
4. （可选）将脚本添加到开机启动：
   - 创建脚本的快捷方式
   - 将快捷方式放入启动文件夹 `%AppData%\Microsoft\Windows\Start Menu\Programs\Startup`

### 自定义配置
- 每个脚本都可以用文本编辑器打开并修改热键
- 修改后保存并重新运行脚本即可生效

## 故障排除

### 脚本不运行
- 确认已安装 AutoHotkey v2.0（检查版本：`AutoHotkey.exe /v`）
- 检查脚本第一行是否包含 `#Requires AutoHotkey v2.0`
- 以管理员身份运行脚本

### 功能无效
- 检查热键是否与其他应用程序冲突
- 确认鼠标侧键正常工作
- 多显示器环境中，确保显示器设置正确

### 性能问题
- 如果 `monitorwiseAltTab.ahk` 响应慢，可能是窗口过多，脚本已优化但仍有极限
- 确保系统有足够内存
